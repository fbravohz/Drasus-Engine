> 🔶 **Enmendado por ADR-0143** — el cómputo sigue siendo Local-First (motor en hardware del usuario), pero deja de ser "sin telemetría": se añade una Cabina de Mando Central del proveedor obligatoria y la soberanía de datos pasa a estar condicionada por el tier de pago (gratis = el proveedor es dueño del trabajo; pago = supresión en origen). Las "Excepciones (Overlays)" de abajo dejan de ser opcionales para el canal de identidad/licencia/telemetría.

### **ADR-0016: Local-First Processing & External Overlays**

*   **Decisión:** Drasus Engine es **Local-First**. El procesamiento de datos, ejecución de estrategias, base de datos y backend residen en la infraestructura local del usuario o su VPS personal.
*   **Excepciones (Overlays):** Se permite el uso de servicios externos (Cloud) exclusivamente para:
    * Autenticación y Gestión de Sesión (Sign-in).
    * Feature Flagging (ej: ConfigCat).
    * Logging/Tracing centralizado opcional.
    * Funcionalidades P2P como Copy Trading.
*   **Justificación:** Evitar costos masivos de infraestructura para el mantenedor del proyecto y garantizar la soberanía de los datos sensibles y el poder de cómputo para el usuario final.

* **Ver también:** ADR-0030 — Concreción de este principio Local-First con decisiones específicas de persistencia (SQLite, DuckDB, Parquet, Zero-Docker). ADR-0134 — Matriz de sistemas operativos donde corre este Local-First (Core nativo en Windows/Linux/macOS y VPS Linux).
