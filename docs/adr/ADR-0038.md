### **ADR-0038: Estándar de Nomenclatura Institucional (Sanitización Terminológica)**

*   **Decisión:** Adoptar de forma obligatoria y exclusiva la nomenclatura formal y técnica definida en la topología de módulos (`ingest`, `generate`, `validate`, `incubate`, `manage`, `execute`, `withdraw`, `feedback`) y en los contextos de superficie de ADR-0136.
*   **Objetivo:** Garantizar el rigor técnico e institucional del proyecto, eliminando términos subjetivos, "gamificados" o niveles de UI deprecados que degradan la autoridad de la infraestructura.
*   **Reglas (Sustituciones Mandatorias):**

    *Módulos y pipeline:*
    * `Mining Rig` / `Mining` → `Ingest` / `Exploración de Alpha`.
    * `Torture Chamber` / `Sala de Torturas` → `Validate` / `Validación de Robustez`.
    * `Autopsia` → `Feedback` / `Análisis de Causa Raíz`.
    * `ADN` (como nombre informal de la lógica de estrategia) → `Genoma` / `Logic Block Editor`.
    * `La Fábrica Visual` → `Canvas Vista Relacional` / `Pipeline Editor`.
    * `El ADN y los Indicadores` → `Canvas Vista Interior` / `Logic Block Editor`.
    * `Fábrica` (genérico) → `Pipeline Editor`.
    * `Autopilot` → `Execute` / `Vigilancia Automática`.

    *Contextos de superficie (ADR-0136 — supersede ADR-0028):*
    * `Fleet Command` (como denominación de nivel ZUI/pantalla) → `Dashboard`.
    * `Orchestrator` (como denominación de nivel ZUI/pantalla) → `Canvas [Forge/Reactor]` / `Canvas Vista Relacional`.
    * `Strategy Inspector` (como denominación de nivel ZUI/pantalla) → `Canvas Vista Interior` / `Inspector Panel lateral`.
    * `MACRO` / `MESO` / `MICRO` (como niveles de interfaz) → contextos de ADR-0136: `Dashboard widget`, `Canvas Vista Relacional`, `Canvas Vista Interior`, `Inspector Panel`.
    * `Nivel 1` / `Nivel 2` / `Nivel 3` (como denominaciones de pantalla ZUI) → ídem anterior.

*   **Implementación:** Auditoría recursiva en toda la documentación (SAD, ADRs, Features, Módulos y TTRs) para eliminar alias prohibidos. Los alias de contexto de superficie (Fleet Command / Orchestrator / Strategy Inspector) se migran progresivamente al construir cada feature en el ROADMAP — ver inventario en `docs/UI-ARCHITECTURE.md §Pendientes`.
*   **Ventaja:** Facilidad de incorporación (onboarding) y consistencia absoluta entre código, base de datos y documentación.
*   **Resultado:** Infraestructura soberana con lenguaje técnico unificado y profesional.
*   **Trazabilidad:** ADR-0136 (Canvas — supersede ADR-0028), `docs/UI-ARCHITECTURE.md`.
