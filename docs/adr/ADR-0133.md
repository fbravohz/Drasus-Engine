### **ADR-0133: Pirámide de Pruebas Canónica + Fuzzing Obligatorio en Fronteras Externas**

*   **Decisión:** Se establece la pirámide de pruebas oficial de Drasus Engine con siete capas ordenadas por prioridad de adopción, se declara `cargo-fuzz` como herramienta obligatoria en todas las fronteras que consumen datos externos no confiables, y se fija la regla de activación del QA-Engineer por fase (EPIC).

*   **Problema que resuelve:** La política de pruebas vigente (2026-06-12) cubría unitarios e integración y mencionaba `proptest` y `criterion` en el skill del QA-Engineer, pero ninguno estaba formalizado como obligatorio ni tenía una regla de cuándo aplicar. El fuzzing era completamente ausente. El resultado: fronteras de parsing externo sin red de seguridad automática y lógica cuantitativa validada solo contra casos de borde que el ingeniero imaginó.

*   **Pirámide canónica (de mayor a menor prioridad de adopción):**

    | Capa | Herramienta | Cuándo es obligatoria | Alcance |
    |---|---|---|---|
    | 1 — Unitarios | `cargo test` (`#[cfg(test)]`) | Siempre, desde EPIC-0 | Lógica pura del dominio (`domain/`): casos válidos, inválidos y de borde |
    | 2 — Integración | `cargo test` (`tests/`) | Siempre, desde EPIC-0 | Orquestación + persistencia; durabilidad usa SQLite en archivo, nunca `:memory:` |
    | 3 — Propiedad | `proptest` | Toda función cuantitativa pura | Invariantes matemáticas ("drawdown siempre ≤ 0", "position nunca NaN", "mismo seed = mismo hash") |
    | 4 — Fuzzing | `cargo-fuzz` | Fronteras declaradas (ver abajo) | Crash/panic/UB en inputs mutados aleatoriamente |
    | 5 — Benchmarks | `criterion` | Desde EPIC-2 (backtest engine) | SLAs de latencia por ruta (ROADMAP §6) |
    | 6 — Adversariales | manual + `proptest` | Bloqueantes pre-dinero real | Look-ahead injection, simulacro de fallo, reconciliación trades |
    | 7 — Flutter (UI) | `flutter test`, `integration_test` | EPIC-8+ | Widget tests y tests de integración de la cáscara delgada |

*   **Fronteras con fuzzing obligatorio (capa 4):**

    | Frontera | Módulo | Activación |
    |---|---|---|
    | Parsers de datos de mercado externos (CSV, JSON, binario) | Ingest | EPIC-1 (primer parser externo) |
    | Deserialización de timestamps y eventos externos | Validate (PIT Validator) | EPIC-2 |
    | Payload de órdenes entrantes (deserialización externa) | Execute | EPIC-3 |
    | Frontera FFI Rust → Flutter (`flutter_rust_bridge`) | Bridge | EPIC-1 (cuando Bridge activo) |

    `cargo-fuzz` requiere toolchain nightly **solo para el crate de fuzzing** (`fuzz/`); el resto del workspace permanece en stable. Cada target mantiene un corpus de seeds mínimo en `fuzz/corpus/<target>/`.

*   **Diferencia clave proptest vs fuzzing (no son equivalentes):**
    *   `proptest`: genera inputs *estructuralmente válidos* con distribuciones controladas para verificar propiedades matemáticas. Detecta errores de lógica.
    *   `cargo-fuzz`: muta inputs *arbitrariamente* (incluidos inválidos y malformados) guiado por cobertura de código. Detecta crashes, panics y comportamiento indefinido.
    Ambas capas son complementarias; ninguna reemplaza a la otra.

*   **Activación del QA-Engineer por fase:**
    *   **EPIC-0:** QA no es gate obligatorio por Story. Disponible para casos puntuales que el Tech-Lead escale.
    *   **EPIC-1 en adelante:** QA es gate obligatorio (Etapa 5 del Tech-Lead) antes de cerrar cualquier Story que toque lógica de dominio.
    *   **Pre-dinero real (cualquier EPIC):** Las Pruebas de Guerra del QA-Engineer skill §3 (look-ahead injection, simulacro de fallo, reconciliación) son bloqueantes de release — sin excepción ni waiver.

*   **Restricciones:**
    *   Prohibido cerrar un TTR que toque una frontera declarada sin su target de fuzzing en verde.
    *   `proptest` es obligatorio para toda función del `domain/` cuyo output sea un valor numérico cuantitativo (ratio, precio, posición, drawdown).
    *   No se crean widget tests ni tests de integración Flutter antes de EPIC-8.
    *   `criterion` no se exige hasta que haya una ruta con SLA medible (EPIC-2+).

*   **Comandos de validación (copy/paste):**
    ```bash
    # Gate normal (todas las fases)
    cargo build && cargo clippy --workspace --all-targets -- -D warnings && cargo test
    cargo llvm-cov --workspace --summary-only

    # Gate fuzzing (cuando el target existe)
    cargo +nightly fuzz run <target_name> -- -max_total_time=60

    # Gate pre-dinero real (añade benchmarks)
    cargo bench --bench <nombre>
    ```
